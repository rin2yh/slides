# テキスト装飾規約

強調の意味と書き方を 1 対 1 で対応させる。「どのタグで書けばよかったっけ」で迷わせない・使い分けを一目で読めるように。

## 太字

- `**text**` → **周囲の文字色を継承した普通の太字**（`<b>` と同義。色は付かない）
  - CSS: `.slidev-layout strong { color: inherit; font-weight: 700 }`
  - `<b>text</b>` も結果は同じ。素の Markdown で書ける `**text**` を基本にする
  - 本文中の語を強調しても、色は周囲と同じ濃色（`#3a4046`）のまま太くなるだけ ＝ 元スライドの設計どおり（青くはならない。青は次節）

## アクセント（青）は構造要素が持つ

青（`var(--dc-accent)`）は「強調」ではなく**構造**に割り当てる。`**` で青くしようとしない。青が出るのは次の専用機構だけ:

- `###` セクションタグ（mono の小ラベル）
- テーブルの `{.accent}` 行（`.claude/rules/tables.md`）
- 端末ブロックの `{badge}` / `{mark}`（`.claude/rules/code-blocks.md`）
- コードの `// [!code ++]` 行
- 目次の番号（`ol` marker）、リンク、セクション番号

「この語だけ青くしたい」という素朴なプローズ強調は設計に無い。青が要るなら上のどれかの構造に落とし込む。

## モノスペース

- `` `code` `` → 灰色モノスペース（inline code）
- ``**`code`**`` → 周囲色の**太字**モノスペース（色は付かない。`**` が普通の太字になったため）
